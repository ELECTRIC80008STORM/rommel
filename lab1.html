Preguntas del Laboratorio 1

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    2. Internet es una red de computadoras interconectadas distribuidas físicamente alrededor del mundo. Por otro lado, la World Wide Web es un sistema de páginas interconectadas accesibles a través de internet. (MDM Web Docs, s.f.).
  2. ¿Cuáles son las partes de un URL?
    3. Una URL se conforma de las siguientes partes (Aviani, 2019):
    Protocolo: Generalmente HTTP o HTTPS.
    Dominio: El nombre del identificador (generalmente una o más direcciónes IP).
    Path: El camino hacia la dirección donde se encuentra el recurso.
    Parámetros: Información adicional que se envía al servidor para identificar o filtrar recursos.
  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    4. El propósito de los siguientes métodos es el siguiente (Aviani, 2019):
    Get: Se usa para pedir datos de un recurso en específico.
    Head: Se usa para lo mismo que get con la diferencia de que no requiere un cuerpo, generalmente ayuda a determinar si el recurso que buscamos está en la página.
    Post: Se usa para enviar información al servidor.
    Put: Se usa para actualizar un recurso que ya estuviera en el servidor.
    Patch: Se usa para aplicar modificaciones parciales a un recurso.
    Delete: Se usa para eliminar un recurso en específico.
  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    5. El método Post ya que este envía los datos al cuerpo del mensaje en lugar de la URL.
  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    6. Se utiliza el método Get, pues este solicita un recurso en específico, y por ende el servidor puede responder con la página web solicitada.
  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    7. El código 200 de respuesta significa que todo salió bien.
  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    8. No sería su responsabilidad, el código de error 400 significa que el recurso buscado no fue encontrado por un error en la parte de quién se comunica con el servidor.
  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    9. En este caso posiblemente sí pues un código de error 500 significa que el servidor fue quien tuvo problemas de poder cumplir la petición.
  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    10. Que un atributo este depreciado significa que existen otros atributos que pueden usarse en su lugar, y que en general, debería dejar de usarse por completo en la medida de lo posible. Algunos métodos depreciados son: <center>, <isindex>, <tt>, <strike>, en general, varias de ellas fueron las relacionadas al manejo de estilos, pues en HTML5 se maneja generalmente por CSS.
  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    11. HTML 5 es la última versión de HTML, y HTML 4 es su antecesor. Sus principales diferencias es que la última versión tiene estructuras más claras de código, facilita la reproducción multimedia y entre varias cosas, añade nuevos elementos semánticos y mejoras a los formularios.
  11. ¿Qué componentes de estructura y estilo tiene una tabla?
    12. Sus componentes estructurales son las filas (<tr>), las columnas (<th>) y el contenido (<td>), las tablas en sí se pueden organizar en (<thead>, <tbody> o <tfoot>). Los de estilo son el tamaño (width y height), bordes (solid, dotted, dashed), espacio interno y externo (padding, margin), alineación del texto (left, right, center o justify), el color de fondo (background-color y color) y las fuentes.
  12. ¿Cuáles son los principales controles de una forma HTML5?
    13. Los principales controles son los siguientes:
    Email: type="email"
    Contraseña: type="password"
    Números: type="number"
    Campo oculto: type="hidden"
  13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?
    14. Yo utilizo Brave, al estar basado en Chrome tiene el mismo nivel de soporte. El cuál no es completo sino parcial, soportando el uso al menos de los elementos más comúnmente usados.
  14. Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?
    15. El ciclo de vida son las series de fases que van desde el inicio (creación) hasta el final del sistema de información. Por otro lado el ciclo de desarrollo únicamente es la serie de fases o etapas que concluyen con la entrega del sistema de información, es decir, únicamente abarcan el proyecto.

Preguntas del Laboratorio 2

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    2. Dado que !important permite cambiar la priorización de estilos, se debe usar con cuidado pues si bien es útil en ciertos casos, puede hacer difícil mantener el código al romper la propiedad de cascada que CSS tiene de manera natural.
  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    3. Porque puede presentar problemas de visibilidad al tener un carácter más fuerte que otros elementos. Es decir, puede hacer ilegible lo demás.
  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    4. Las unidades correctas pueden facilitar ciertas tareas, su relación es la siguiente:
    % Sirve para un diseño más responsivo pues se adapta al tamaño del contenedor.
    px Es mejor para una mayor precisión y control.
    pt Puede usarse mejor para imprimir.
  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    5. Mejora el rendimiento al reducir el tamaño del archivo CSS, un mejor rendimiento significa al menos un menor tiempo de carga, algo visto de manera positiva por los usuarios.
